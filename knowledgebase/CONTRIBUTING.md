# Contributing Guidelines

## Overview

This document provides guidelines for contributing to the Board Game Borrow documentation. Following these guidelines helps maintain consistency and quality across our documentation.

## Documentation Structure

### File Organization
- Keep files in appropriate directories
- Use clear, descriptive filenames
- Follow existing naming conventions

### Content Structure
- Use clear headings and subheadings
- Include table of contents for longer documents
- Maintain consistent formatting

## Writing Style

### General Guidelines
- Be clear and concise
- Use active voice
- Write in present tense
- Keep paragraphs focused and brief

### Code Examples
- Include relevant code snippets
- Use proper syntax highlighting
- Add comments for clarity
- Show complete, working examples

## Making Changes

### Process
1. Create a new branch
2. Make your changes
3. Test documentation links
4. Submit pull request
5. Address review feedback

### Commit Messages
- Use clear, descriptive messages
- Reference related issues
- Follow conventional commits format

## Document Types

### Technical Docs
- Include setup instructions
- List prerequisites
- Provide troubleshooting guides
- Add code examples

### Architecture Docs
- Include diagrams
- Explain system interactions
- Document design decisions
- Update when architecture changes

### API Docs
- Document all endpoints
- Include request/response examples
- List error codes
- Show authentication methods

## Review Process

### Before Submitting
- Check for technical accuracy
- Verify all links work
- Run spell check
- Test code examples

### Review Criteria
- Technical accuracy
- Clarity and completeness
- Proper formatting
- Code correctness

## Best Practices

### Documentation
- Keep docs up to date
- Remove outdated content
- Add relevant examples
- Include error handling

### Markdown
- Use proper heading levels
- Include code blocks
- Add lists where appropriate
- Use tables for structured data
